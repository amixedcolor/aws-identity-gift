1. 診断の「方針」に応じたプロンプトの味付け案
ユーザーが選んだ方針によって、AI（Claude 4.5）が「なぜあなたにこのサービスを贈るのか」の語り口を変えます。

A. スキル・論理重視（Tech-Fit型）
アプローチ: ユーザーの「普段のコードの書き方」や「パズルの解き方」から、技術的な納得感が高いサービスを提案。
例: * 「几帳面で型定義をしっかり書きたい」 → AWS Step Functions （状態遷移を完璧にコントロールする快感をプレゼント）
「とにかく速く結果を出したい」 → AWS Amplify （爆速で形にする全能感をプレゼント）
→そうではなく、既存のスキルや経験を見るようにしたいです。どのAWSサービスをどれくらい経験しているのか、そこから探る感じ。

B. 性格・ライフスタイル重視（Vibe-Fit型）
アプローチ: ユーザーの「休日の過ごし方」や「好きな家事」「人間関係のスタンス」から、価値観が合うサービスを提案。
例:
「キャンプが好きで、最小限の道具で工夫するのが好き」 → AWS Lambda （身軽で無駄のない美学をプレゼント）
「掃除好きで、散らかったものを整理するのが得意」 → Amazon S3 / Lake Formation （究極の整理整頓術をプレゼント）
→これはイメージ通り

C. 憧れ・意外性重視（Adventure型）
アプローチ: ユーザーが「まだ知らない世界」や「自分にはない要素」への興味から、新しい可能性をプレゼント。
例:
「SF映画や宇宙が好き」 → AWS Ground Station （地上を飛び出し、宇宙と繋がるロマンをプレゼント）
「実は量子力学とか不思議な話が好き」 → Amazon Braket （次世代の知的好奇心をプレゼント）
→これもイメージ通り。存分に尖らせて、奇をてらっていこう。

2. クイック診断（1分）のための質問案
これらを「クリスマスらしい、答えやすい質問」に落とし込みます。
プレゼントを包むなら？
A: 角までキッチリ、美しく包みたい（論理派）
B: 中身がすぐ取り出せるように簡易的に包む（効率派）
C: 開けた時の驚きを重視して豪華に飾りたい（演出派）
冬休みの宿題、どう進めるタイプだった？
A: 計画を立てて毎日コツコツ（オーケストレーション型）
B: 最後に一気に、超集中して終わらせる（サーバーレス・イベント駆動型）
C: そもそも宿題をハックして楽する方法を考える（マネージドサービス活用型）
次に旅行へ行くなら、どこで何をしたい？
（自由記述 or 選択肢：賑やかな都会、静かな雪山、誰もいない秘境）
→クリスマスに関連させるのいい感じだけど、これは「B. 性格・ライフスタイル重視（Vibe-Fit型）」でしか使え無さそう？方針に応じて別個に適した質問があったほうがいいかも。

3. 実装に向けたヒアリング
UI/UXを具体化するために、以下の2点について伺わせてください。

① 「セッション保存」の表示形式
LocalStorageに保存された「過去のプレゼント」は、どのように表示したいですか？
（例）画面の隅に「過去に受け取ったプレゼントボックス」が並んでいて、クリックするとその時の診断結果が再表示される。
（例）タイムライン形式で、いつどんなサービスを贈られたかが履歴として残る。
→クリスマスツリーのフリー画像の近くに、プレゼントが増えていく感じ。クリックすると再表示。タイムライン形式は不要だが、再表示時にタイムスタンプ的なのは見せたい（いつの診断か）

② 生成AIに「どこまで」任せるか
パターン1: AWSサービス名の選定自体をAIに丸投げする。
パターン2: あらかじめ30個ほど「プレゼント候補のAWSサービス」のリストをこちらで用意しておき、AIはその中から最適なものを選び、「なぜあなたにこれなのか」のレター（解説文）を書く。
※精度と「代名詞感」を出すなら、パターン2の方が外れがなく、個別のメッセージにAIのパワーを割けるのでおすすめです。
→パターン2にする。ただし、サービスのリストは100個くらいでもいいかも？（いまトータルで200個ほどあるはず）
